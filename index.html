Intelligent Components

New Design

Skin has been built from the ground up to support eBay's new mobile design system. Skin will evolve as the design system evolves, meaning you only need to keep your Skin package updated to ensure the latest look and feel.

Mobile-First

All screen sizes 7" and below are considered to be small screens; they are our default, baseline experience. All screen sizes over 7" are considered to be large screens; Skin provides support for these large screens in an adaptive manner.

Accessibility

Accessibility is at the heart of Skin. A widget must not only look like a certain type of widget, it must be described like that widget too. Skin leverages ARIA roles, states and properties to apply our styles wherever possible, thus enforcing correct, accessible semantics.

Pure CSS

Skin is exclusively a CSS toolkit, meaning that JavaScript is not included as part of this project. Remaining decoupled from the JavaScript layer means that our HTML and CSS should not need to be reimplemented every time the JS frontend framework changes.

BEM

Skin follows the BEM principles of Block, Element and Modifier to ensure our class name and class structure is human readable and understandable. The princicples of BEM also act as a front-line defense against classitis.

Open Source

Skin operates under an open source model. If you wish to contribute, raise an issue, or ask a question, head over to GitHub for more information.

Install

Skin is distributed as an NPM package. Install the package using the npm install command.

npm install skin-ebay --save

Include the desired Skin modules in your browser.json and you are good to go.

{
    "dependencies": [
        "skin-ebay/core",
        "skin-ebay/button"
    ]
}

Legacy Browsers

Skin takes advantage of Lasso's conditional dependencies to provide cross-browser support for legacy browsers (e.g. browsers that do not support Flexbox or SVG). To plugin this conditional dependency, simply pass the flag skin-legacy to Lasso and apply the skin-legacy class to the body of your page. Skin itself does not do any detection of browser or browser capabilities, therefore it is the responsibility of the application logic to detect whether the legacy styles are needed.

Button

Buttons are used to trigger form actions (e.g. submit, reset) or JavaScript actions (e.g. open an overlay or menu). There are two tags for creating buttons: <button> (supports inner HTML) and <input> (supports inner text only). Outside of forms, the <button> tag is always recommended. The only button that can update the full URL is a form submit button.

When using the <button> tag, remember to always specify it's type attribute (type="submit" is the default, if not specified).

Links are used exclusively to navigate URLs and therefore the href attribute must always contain a valid URL ('javascript' is not a valid URL so please instead use a button for javascript behaviour). Only the <a> tag can be used to create links. Links always trigger a GET request, which means that they should always be idempotent. If you wish to trigger an action that chages data on the server, then you should use a form submit to trigger a POST request.

Default Button

To create a default button, use class .btn.

<button class="btn" type="button">Buy it Now</button>

Primary Button

To indicate that a button represents a primary action, use the .btn--primary modifier class.

<button class="btn btn--primary" type="button">Buy it Now</button>

Secondary Button

To indicate that a button represents a secondary action, use the .btn--secondary modifier class.

<button class="btn btn--secondary" type="button">Buy it Now</button>

Small Button

For smaller buttons use the .btn--small modifier class.

<button class="btn btn--small" type="button">Buy it Now</button>

Medium Button

The default button size is medium and no modifier class in needed.

<button class="btn" type="button">Buy it Now</button>

Large Button

For larger buttons, use the .btn--large modifier class.

<button class="btn btn--large" type="button">Buy it Now</button>

Fluid Button

A fluid button fills the entire width of it's parent element. Please use the .btn--fluid modifier class.

<button class="btn btn--primary btn--fluid" type="button">Fluid Button</button>

Button Group

Wrap a series of buttons inside a .btn-group component.

<div class="btn-group">
    <button class="btn btn--primary btn--small" type="button">Add to cart</button>
    <button class="btn btn--primary btn--small" type="button">Buy It Now</button>
</div>

Disabled Button

To disable a button, apply the disabled attribute.

<button class="btn" type="button" disabled>Buy it Now</button>

Icon Buttons

Skin provides a fixed set of icon buttons that can be used by simply by applying the relevant modifier class. For example btn--cart or btn--sort.

<button class="btn btn--small btn--cart" type="button">Add To Cart</button>
<button class="btn btn--small btn--sort" type="button">Sort</button>

Custom Icon Buttons

If you do not find the icon button you need, you can either ask us to add it to our set of button classes, or you can create it yourself by leveraging our icon classes.

<button class="btn btn--small icon-close" type="button">Close</button>
<button class="btn btn--small icon-menu" type="button">Menu</button>

Icon Buttons and Accessibility

An icon on a button that contains no visible text, is always deemed a critical icon. An aria-label must be provided for assistive technology to identify the purpose of the control.

<button class="btn icon-close" aria-label="Close" type="button"></button>
<button class="btn icon-menu" aria-label="Menu" type="button"></button>
<button class="btn icon-info" aria-label="Information" type="button"></button>

Submit & Reset

For form submit & reset buttons, use class .btn and any desired modifiers.

<input class="btn" type="submit" value="Submit" />
<input class="btn" type="reset" value="Reset" />

Link

A regular text link is styled by the Core module and does not need any classes.

Visit Store 
<a href="http://www.ebay.com">Visit Store</a>

Faux Button

Links can be styled to look like buttons, whilst still behaving like links. You should only do this for prominent 'call to action' type links, e.g. 'Start Shopping', 'Take me There', etc.

Please use the btn--faux modifier class in this case. The href attribute must always contain a valid URL.

Shop Now 
<a class="btn btn--faux" href="http://www.ebay.com">Shop Now</a>

Disabled Faux Button

To mark an anchor tag as disabled, apply the aria-disabled state. Apply a negative tabindex to prevent keyboard focus. Use javascript to prevent default click behaviour.

Disabled 
<a class="btn btn--faux" href="http://www.ebay.com" aria-disabled="true" tabindex="-1">Disabled</a>

Link Hijax

Using JavaScript, a link can have it's URL navigation behaviour progressively enhanced to button behaviour. For example, a link that would take the user to a new page might be converted to launch a view of the new page inside of a dialog instead. This technique is often referred to as 'hijax'.

Please use the btn--hijax modifier class in this case. The href attribute must always contain a valid fallback URL. A role of button must also be specified.

View Item 
<a class="btn btn--hijax" href="http://www.ebay.com" role="button">View Item</a>

Core

Core centralizes common functionality that is re-used by other modules. It includes the following sub-modules: normalize, typography and atoms.

Normalize

Normalize makes browsers render all elements more consistently and in line with modern standards. Unlike a reset.css normalize.css will not reset browser default styles, it provides instead a default baseline to achieve cross browser consistency. If you want to know more about the differences between normalize.css and reset.css check this answer on stackoverflow.

Typography

Typography creates a unique look for the eBay brand by defining the style of common components such as headings, paragraphs and links.

Atoms

Atoms are a small collection of drop-in utility classes. Some atoms are applied directly by our Experience Service to represent business logic (e.g. item-hotness). Some classes that you would typically apply frequently in your JavaScript logic (e.g. hide and show) are also provided for your conveniance. Finally, in some cases, you might apply these atoms directly to your HTML template (e.g. clearfix and clipped).

Class Properties Class Properties
.hide display: none; .show display: block;
.left float: left; .right float: right;
.center margin:0 auto; .no-scroll overflow: hidden !important
.clearfix Clear floats .clipped Element is clipped and visible to screen reader only
.image-wrapper Vertically and horizontally center an image. .truncate Truncate single-line text.
.emphasis color: #DD1E31; .positive color: #1F9900;
.negative color: #ED2819;

Dialog

A dialog is a child window that is spawned by the main web page or application. Skin dialogs are always modal, meaning that interactions with the dialog's parent window must be prevented while the dialog is open.

A dialog must always be labelled using the aria-labelledby or aria-label properties.

Dialog Transitions

Dialog hide/show transitions are triggered when aria-hidden state is toggled. A JavaScript timeout function is required to correctly trigger CSS transitions, see commented example code below for further details.

$('#dialog-open').on('click', function() {
    // immediately set display to block to enable CSS transitions
    $('.dialog').css('display', 'block');

    // set aria-hidden state to trigger opening transition
    setTimeout(function() {
        $('.dialog').attr('aria-hidden', 'false');
    }, 10);
});

$('#dialog-close').on('click', function() {
    // immediately set aria-hidden state to trigger closing transition
    $('.dialog').attr('aria-hidden', 'true');

    // after close transition has finished, it is safe to set display to none
    setTimeout(function() {
        $('.dialog').css('display', 'none');
    }, 375);
});

Note: The transition duration varies for each type of dialog.

Default Dialog

The default dialog is a full-page like experience on small screens, and a center-aligned window for large screens. For large screens, a fixed height is required on the div with role=document.

Transition duration: 150ms.

Notifications

The dialog content goes here.

<div role="dialog" class="dialog" aria-labelledby="dialog-0-title" aria-hidden="true">
    <div role="document">
        <header>
            <button aria-label="Close Dialog" type="button"></button>
            <h2 id="dialog-0-title">Notifications</h2>
        </header>
        <div>
            <!-- dialog content goes here -->
            <p>The dialog content goes here.</p>
        </div>
    </div>
</div>

Sub-Page Dialog

Sub-Page Dialog is a variant of the default modal; it is invoked from a right-facing chevron, and is apt for the mobile hub-and-spoke model.

Sub-Page Dialog slides in from the right side of the screen and is intended for small screens only.

Transition duration: 350ms.

Product Details

The dialog content goes here.

<div role="dialog" class="dialog dialog--subpage" aria-labelledby="dialog-1-title" aria-hidden="true">
    <div role="document">
        <header>
            <button aria-label="back" type="button"></button>
            <h2 id="dialog-1-title">Product Details</h2>
        </header>
        <div>
            <!-- dialog content goes here -->
            <p>The dialog content goes here.</p>
        </div>
    </div>
</div>
<div class="mask"></div><!-- /mask -->

Panel Dialog

Panel Dialog offers additional UI controls and should not completely draw the user’s attention from the current task.

Panel Dialog slides in from the left of the screen to fill 80% of the available space, and is intended for small screens only.

Transition duration: 350ms.

<div role="dialog" class="dialog dialog--panel" aria-label="panel" aria-hidden="true">

        <header>
            <button aria-label="Close Dialog"></button>

        </header>

            <!-- dialog content goes here -->
        </div>
    </div>
</div>

Form

A form displays a set of fields that allow users to input data. Fields include: textboxes, listboxes, radios, checkboxes and switches. Fields can have various different properties and states.

Assistive technology requires accessible labels for all form controls, without exception. In some rare circumstances, labels can be visually hidden - read on to find out more.

Textbox

A textbox can be single-line or multi-line, using the input or textarea element, respectively. By default, textbox labels will always appear above the input field (i.e. the labels are set to 'block' level).

Default Textbox

<div class="textbox">
    <label for="textbox-example-1">Email Address</label>
    <input id="textbox-example-1" type="email" name="email" />
</div>

Multi-line Textbox

A multi-line textbox allows line breaks.

<div class="textbox">
    <label for="textbox-example-2">Description</label>
    <textarea id="textbox-example-2" name="description"></textarea>
</div>

Inline Textbox

A textbox with label beside it. NOTE: Label length is restricted to 25 characters.

<div class="textbox textbox--inline">
    <label for="phonenumber">Phone Number</label>
    <input id="phonenumber" type="phonenumber" placeholder="Phone Number" name="phonenumber" />
</div>

Textbox Placeholders

Placeholder text is intended as a short hint towards the type of value that should be entered. Both single-line and multi-line textboxes can have placeholder text. Placeholder text alone does not count as an accessible label, because this text is transient (i.e. it gets cleared on user input).

<div class="textbox">
    <label for="placeholder-example">Email Address</label>
    <input id="placeholder-example" type="email" placeholder="yourname@company.com" name="email" />
</div>

Textbox Hidden Label

The aria-label property can be used to provide a non-visual label that remains accessible to assistive technology. To prevent possible cognitive difficulties for our users, a hidden label is only acceptable if there is some other visual clue (e.g. an icon) as to the purpose of the field.

Remember too that placeholders are not an accessible alternative to visual labels, the only possible exception to this rule is for very short forms (e.g. a login form, as in the example below), and even then a hidden label is still always required.

<div class="textbox">
    <input type="text" placeholder="Username" name="username" aria-label="Username" />
</div>
<div class="textbox">
    <input type="password" placeholder="Password" name="password" aria-label="Password" />
</div>

Textbox Icons

Skin provides a set of textbox icons that can be used simply by applying the relevant modifier class to the input. For example textbox--username or textbox--search. These icons are currently only compatible with single-line textboxes. If you do not find the textbox icon that you need, let us know and we will consider adding it to our set.

<div class="textbox textbox--username">
    <input type="text" name="username" aria-label="Username" placeholder="Username" />
</div>
<div class="textbox textbox--search">
    <input type="text" name="search" aria-label="Search" placeholder="Search" />
</div>

Small Textbox

For a smaller textbox (including any label and icon), apply the .textbox--small modifier class.

<div class="textbox textbox--username textbox--small">
    <input type="text" name="username" aria-label="Username" placeholder="Username" />
</div>
<div class="textbox textbox--search textbox--small">
    <input type="text" name="search" aria-label="Search" placeholder="Search"  />
</div>

Readonly Textbox

A readonly field is conveyed using the readonly attribute. Readonly field values do get passed to the server. Note that the readonly attribute has no effect on a listbox.

<div class="textbox">
    <label for="readonly-example">Email Address</label>
    <input id="readonly-example" type="text" value="yourname@company.com" name="readonly-example" readonly />
</div>

Listbox

Allows the user to select one item from a list of options.

A listbox (also known as a 'select') is typically used to collate form data and therefore must always be used in conjunction with a form submit button. If you are not submitting form data or do not wish to reload the entire page, then a Menu maybe a better choice.

Default Listbox

By default, listboxes encourage one of the options to be auto-selected using the selected property. This is known as a sensible default.

<div class="listbox">
    <label for="size-selection">Choose a size</label>
    <select id="size-selection">
        <option value="1" selected>Small</option>
        <option value="2">Medium</option>
        <option value="3">Large</option>
        <option value="4">Extra Large</option>
    </select>
</div>

Unselected Listbox

To force a user to make an actual selection themselves, the default option can be used as a prompt and must be set to disabled.

<div class="listbox">
    <label for="size-selection">Choose a size</label>
    <select id="size-selection">
        <option value="0" disabled selected>-select-</option>
        <option value="1">Small</option>
        <option value="2">Medium</option>
        <option value="3">Large</option>
        <option value="4">Extra Large</option>
    </select>
</div>

Unlabelled Listbox

For listboxes without a visible label, use aria-label to provide an accessible label to assistive technology. The listbox--no-label modifier class must also be provided.

<div class="listbox listbox--no-label">
    <select id="size-selection" aria-label="Choose a size">
        <option value="0" disabled selected>Choose a size</option>
        <option value="1">Small</option>
        <option value="2">Medium</option>
        <option value="3">Large</option>
        <option value="4">Extra Large</option>
    </select>
</div>

Switch

A switch behaves very much like a checkbox - it can be on or off (i.e checked or unchecked) - and under the hood, that's exactly what it is. Whereas checkboxes are often used to allow multi-selection from a group of choices, switches are more often used in isolation or as a series of unrelated options.

<div class="switch">
    <label for="toggle-switch">Returns Accepted</label>
    <input id="toggle-switch" type="checkbox" name="toggle-switch" />
    <span></span>
</div>

Disabled Switch

A toggle switch can be disabled using the disabled property.

<div class="switch">
    <label for="toggle-switch-disabled">Returns Accepted Disabled</label>
    <input id="toggle-switch-disabled" type="checkbox" name="toggle-switch-disabled" disabled />
    <span></span>
</div>

Form States & Properties

Error State

An error is conveyed using the aria-invalid state. Additional context for the error is conveyed using the aria-describedby property which must point to the unique id of the error message container. This error message container can be designated as a live region if client-side input validation is implemented and enabled.

Please enter a valid email address

<div class="textbox">
    <label for="error-example">Email Address</label>
    <input id="error-example" type="email" name="email" aria-invalid="true" aria-describedby="invalid-email" value="foobar" />
    <div aria-live="polite" id="invalid-email">
        <p>Please enter a valid email address</p>
    </div>
</div>

Disabled State

A disabled field is conveyed using the disabled attribute. Disabled field values are not passed to the server.

<div class="textbox">
    <label for="disabled-textbox">Email Address</label>
    <input id="disabled-textbox" type="text" value="yourname@example.com" name="disabled-textbox" disabled />
</div>
<div class="listbox listbox--no-label listbox--disabled">
    <label for="disabled-listbox" class="clipped">Choose a Size</label>
    <select id="disabled-listbox" disabled>
        <option value="0">Choose a Size</option>
        <option value="1">Small</option>
        <option value="2">Medium</option>
        <option value="3">Large</option>
        <option value="4">Extra Large</option>
    </select>
</div>

Readonly State

A readonly field is conveyed using the readonly attribute. Readonly field values do get passed to the server. Note that the readonly attribute has no effect on a listbox.

<div class="textbox">
    <label for="readonly-example">Email Address</label>
    <input id="readonly-example" type="text" value="yourname@company.com" name="readonly-example" readonly />
</div>

Required Property

A required field is conveyed visually using an asterisk in superscript, and for assistive technology using the aria-required property.

<div class="textbox">
    <label for="email">Email Address <sup>*</sup></label>
    <input id="email" type="email" placeholder="yourname@company.com" name="email" aria-required="true" />
</div>

<div class="form form--small">

Icon

Skin 2.0 uses icon fonts. Although icon fonts are supported across all modern browsers, they sometimes experience "weird failures". Please refer to this GitHub issue, which summarizes the failure scenarios and provides fallback options.

Important: The classname prefix of icon- is reserved by Skin. Please do not use it anywhere in your application CSS class names.

Icons with Text

Icons with adjacent text are deemed to be non-critical if the text adequately conveys the same message as the icon. Non-critical icons should be hidden from assistive technology using the aria-hidden state.

We had trouble connecting to PayPal. Please make your payment later.

<div class="demo">
    <p>
        <span class="icon-alert" aria-hidden="true"></span>
        We had trouble connecting to PayPal. Please make your payment later.
    </p>
</div>

Icons without Text

An icon with no adjacent text may be deemed critical or non-critical to assistive technology, depending on whether it conveys important information or not.

Icons deemed critical must provide alternative text using aria-label and must specify role=img.

<span class="icon-alert" aria-label="Alert" role="img"></span>

Note that in the case of buttons, and other interactive controls, an ARIA state or property might also convey the same information as the visual icon, e.g. a chevron icon and aria-expanded or a sort icon and aria-haspopup, in which case the icon is no longer deemed critical.

Icons without text that are deemed non-critical should be hidden from assistive technology using the aria-hidden state.

<span class="icon-images-null" aria-hidden="true"></span>

Spinner

A spinner animation is used to convey a busy or loading state, often as the result of a user action.

A spinner is considered a critical icon, therefore an img role and aria-label property are required.

<span class="spinner" aria-label="Busy animation" role="img"></span>

Large Spinner

Use the spinner--large modifier class to create a large spinner.

Menu

A menu follows the principal of progressive disclosure by keeping advanced functionality hidden from view until it's button is activated.

When adding a menu to the page, you have two choices: selecting a menu item can either update the page without a full page reload, or it can reload the entire page (we call this our Faux Menu); be sure to use the correct version for your needs!

NOTE: Menus are not form controls. If you wish to submit form data, please consider Listbox instead.

Default Menu

Apply role menuitem on each menu item.

Zoom out

Tab

A tabbed interface allows the layered stacking of content panels, with only one panel being visible at any time. A tab is contained within a tablist - each tab controlling the visibility of it's respective panel. By decluttering the user-interface in this way, we say that a tab follows the principals of progressive disclosure.

When adding a tabbed interface to a page, you have two choices: selecting a tab can either update the panel contents dynamically, without a full page reload (these are our default tabs), or statically, reload the entire page (we call these our 'Faux' Tabs) - be sure to use the correct version for your needs!

Default Tabs

When a tab is selected, the aria-selected, aria-hidden and tabindex states must be updated in order for the CSS to reflect the change. Only one tab can be selected and in the tab order at any moment in time. Likewise, only one tabpanel can be visible at any time, and it must correspond to the currently selected tab.
