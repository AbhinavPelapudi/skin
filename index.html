Introduction

Skin 2.0 has been rebuilt from the ground up to support eBay's new Vision Quest design system for mobile and is not backwards compatible with earlier versions.

Skin 2.0 is exclusively a CSS toolkit. This means that JavaScript behaviour is not included as part of this project. Remaining decoupled from the JavaScript layer means that HTML and CSS should not need to be reimplemented every time the JS frontend framework changes.

Skin 2.0 operates under an open source model. You can find the code in GitHub. If you wish to contribute feel free to issue a pull request, but we would advise you to talk with us first, either through slack at ebayui.slack.com or by sending an email to DL-ebay-skin@ebay.com. Finally if you want to file a bug, you can do so in the issues section.

What's new in Skin 2.0

Accessibility

Accessibility is at the heart of Skin. A widget must not only look like a certain type of widget, it must be detected and described like that widget too. Skin 1.x relied solely on classes to apply visual styles, but classes do not convey any semantic meaning to assistive technology (or search engines). In Skin 2 we now leverage fully semantic HTML and ARIA roles, states and properties to apply our styles, thus enforcing correct, accessible semantics.

Mobile

With mobile traffic steadily increasing, we have built this version with a mobile first approach. Tablet and desktop breakpoints are not currently supported.

Variables & Mixins

Skin exposes its core LESS variables & mixins as a separate module called skin-ebay/less. Including this module as a dependency, will enable developers to access Skin LESS variables & mixins in their application. Please refer to the Getting Started section on adding dependencies.

Core

The core module centralizes common functionality that is re-used by other modules. It includes the following sub-modules: normalize, typography, atoms and mixins.

Normalize

Normalize is a CSS file that makes browsers render all elements more consistently and in line with modern standards. Unlike a reset.css normalize.css will not reset browser default styles, it provides instead a default baseline to achieve cross browser consistency. If you want to know more about the differences between normalize.css and reset.css check this answer on stackoverflow.

Typography

Typography creates a unique look for the eBay brand by defining the style of common components such as paragraphs, links and headers.

Atoms

Our atoms are a small collection of drop-in utility classes. Some atoms are applied directly by our Experience Service to represent business logic (e.g. item-hotness). Some classes that you would typically apply frequently in your JavaScript logic (e.g. hide and show) are also provided for your conveniance. Finally, in some cases, you can apply these atoms directly to your HTML template (e.g. clearfix and clipped).

Mixins

Advanced users may want to work directly with the .less files, in this case Core also provides a collection of usefull mixins.less and variables.less.

Buttons & Links

Buttons are used to trigger form actions (e.g. submit, reset) or JavaScript actions (e.g. open overlay or menu). There are two tags for creating buttons: <button> (supports inner HTML) and <input> (supports inner text only). Outside of forms, the <button> tag is always recommended. The only button that can update the URL is a form submit button.

Links are used exclusively to navigate URLs and therefore the HREF attribute must always contain a valid URL ('javascript' is not a valid HREF so please use a button instead). There is only one tag to create links: <a>. Links always trigger a GET request, which means that they should always be idempotent. If you wish to trigger an action that chages data on the server, then you should use a form submit to trigger a POST request.

Default Button

To create a default button, use class .btn. When using a <button> element, remember to specify it's type attribute.

<button class="btn" type="button">Buy it Now</button>

Primary Button

To indicate that a button represents a primary action, use the .btn--primary modifier class name.

<button class="btn btn--primary" type="button">Buy it Now</button>

Secondary Button

To indicate that a button represents a secondary action, use the .btn--secondary modifier classname.

<button class="btn btn--secondary" type="button">Buy it Now</button>

Small Button

For smaller buttons use the .btn--small modifier classname.

<button class="btn btn--small" type="button">Buy it Now</button>

Medium Button

The default is medium size buttons .btn.

<button class="btn" type="button">Buy it Now</button>

Large Button

For larger buttons, use the .btn--large modifier classname.

<button class="btn btn--large" type="button">Buy it Now</button>

Fluid Button

A fluid button fills the entire width of it's parent element.

<button class="btn btn--primary btn--fluid" type="button">Fluid Button</button>

Button Group

Wrap a series of buttons by nesting .btn elements inside a .btn-group.

<div class="btn-group">
    <button class="btn btn--primary btn--small" type="button">Add to cart</button>
    <button class="btn btn--primary btn--small" type="button">Buy It Now</button>
</div>

Disabled Button

To disable a button, add the disabled attribute.

<button class="btn" disabled>Buy it Now</button>

Icon Buttons

Skin provides a fixed set of icon buttons that can be used by simply by applying the relevant modifier class name. For example btn--cart or btn--sort.

<button class="btn btn--primary btn--small btn--cart">Add To Cart</button>
<button class="btn btn--small btn--sort">Sort</button>

Custom Icon Buttons

If you do not find the icon button you need, you can either ask us to add it to our set of button classes, or you can create it yourself by leveraging our icon classes.

<button class="btn btn--small icon-close">Close</button>
<button class="btn btn--small icon-menu">Menu</button>

Icon Buttons and Accessibility

An icon on a button that contains no visible text, is always deemed a critical icon. An aria-label must be provided for assistive technology to identify the purpose of the control.

<button class="btn icon-close" aria-label="Close"></button>
<button class="btn icon-menu" aria-label="Menu"></button>
<button class="btn icon-info" aria-label="Information"></button>

Link

A regular Link is styled by the Core module and does not need any classes.

Visit Store 
<a href="http://www.ebay.com">Visit Store</a>

Button Styled Link

Links can be styled to look like buttons, whilst still behaving like links. You should only do this for prominent 'call to action' type links, e.g. 'Start Shopping', 'Take me There', etc.

Shop Now 
<a class="btn" href="http://www.ebay.com">Shop Now</a>

Hijaxed Link

Using JavaScript, a link can have it's URL navigation behaviour enhanced to button behaviour. For example, a view item link that would take the user to the view item page might be converted to a view item button that launches the view item dialog instead. This technique is often referred to as 'hijax'.

View Item 
<a class="btn" href="http://www.ebay.com" role="button">View Item</a>

Disabled Link

To disable a link, remove the href attribute and apply the .btn--disabled class.

Disabled 
<a class="btn btn--primary btn--disabled">Disabled</a>

Dialog

A dialog is a modal overlay that temporarily blocks interactions with the main view of a site.

A dialog must be properly labelled using the aria-labelledby or aria-label properties.

Use the aria-hidden state to toggle the dialog visibility for all users.

Mobile Full width Dialog

Mobile web dialogs are a full-page like experience. The width is set to 100%.

Tablet Dialog

Set max-width of 800px for tablet web dialogs.

Title

The dialog content goes here.

<div role="dialog" class="dialog" aria-labelledby="dialog-label" aria-hidden="false">
    <div role="document">
        <div>
            <button class="dialog__close" aria-label="Close Dialog"></button>
            <h2 id="dialog-label">Title</h2>
        </div>
        <div>
            <!-- dialog content goes here -->
            <p>The dialog content goes here.</p>
        </div>
    </div>
</div>

Forms

A form displays a set of fields that allow users to input data. Fields include: textboxes, listboxes, radios, checkboxes and switches. Fields can have various different properties and states.

Assistive technology requires accessible labels for all form controls, without exception. In some rare circumstances, labels can be visually hidden - read on to find out more.

Textbox

A textbox can be single-line or multi-line, using the input or textarea element, respectively. By default, textbox labels will always appear above the input field (i.e. the labels are set to 'block' level).

Single-line Textbox

<div class="textbox">
    <label for="textbox-example-1">Email Address</label>
    <input id="textbox-example-1" type="email" name="email" />
</div>

Multi-line Textbox

A multi-line textbox allows line breaks.

<div class="textbox">
    <label for="textbox-example-2">Description</label>
    <textarea id="textbox-example-2" name="description"></textarea>
</div>

Textbox Placeholders

Placeholder text is intended as a short hint towards the type of value that should be entered. Both single-line and multi-line textboxes can have placeholder text. Placeholder text alone does not count as an accessible label, because this text is transient (i.e. it gets cleared on user input).

<div class="textbox">
    <label for="placeholder-example">Email Address</label>
    <input id="placeholder-example" type="email" placeholder="yourname@company.com" name="email" />
</div>

Hidden Textbox Label

The aria-label property can be used to provide a non-visual label that remains accessible to assistive technology. To prevent possible cognitive difficulties for our users, a hidden label is only acceptable if there is some other visual clue (e.g. an icon) as to the purpose of the field.

Remember too that placeholders are not an accessible alternative to visual labels, the only possible exception to this rule is for very short forms (e.g. a login form, as in the example below), and even then a hidden label is still always required.

<div class="textbox">
    <input type="text" placeholder="Username" name="username" aria-label="Username" />
</div>
<div class="textbox">
    <input type="password" placeholder="Password" name="password" aria-label="Password" />
</div>

Textbox Icons

Skin provides a set of textbox icons that can be used simply by applying the relevant modifier class to the input. For example textbox--username or textbox--search. These icons are currently only compatible with single-line textboxes. If you do not find the textbox icon that you need, let us know and we will consider adding it to our set.

<div class="textbox textbox--username">
    <input type="text" name="username" aria-label="Username" placeholder="Username" />
</div>
<div class="textbox textbox--search">
    <input type="text" name="search" aria-label="Search" placeholder="Search" />
</div>

Small Textbox

For a smaller textbox (including any label and icon), apply the .textbox--small modifier class.

<div class="textbox textbox--username textbox--small">
    <input type="text" name="username" aria-label="Username" placeholder="Username" />
</div>
<div class="textbox textbox--search textbox--small">
    <input type="text" name="search" aria-label="Search" placeholder="Search"  />
</div>

Listbox

Allows the user to select one item from a list of options.

A listbox (also known as a 'select') is typically used to collate form data and therefore must always be used in conjunction with a form submit button. If you are not submitting form data or do not wish to reload the entire page, then a Menu maybe a better choice.

Listbox with Default Selection

By default, listboxes encourage one of the options to be auto-selected using the selected property. This is known as a sensible default.

<div class="listbox">
    <label for="size-selection">Choose a size</label>
    <select id="size-selection">
        <option value="1" selected>Small</option>
        <option value="2">Medium</option>
        <option value="3">Large</option>
        <option value="4">Extra Large</option>
    </select>
</div>

Listbox with Forced Selection

To force a user to make an actual selection themselves, the default option can be used as a prompt and must be set to disabled.

<div class="listbox">
    <label for="size-selection">Choose a size</label>
    <select id="size-selection">
        <option value="0" disabled selected>-select-</option>
        <option value="1">Small</option>
        <option value="2">Medium</option>
        <option value="3">Large</option>
        <option value="4">Extra Large</option>
    </select>
</div>

Listbox with no Visible Label

For listboxes without a visible label, use aria-label to provide an accessible label to assistive technology. The listbox--no-label modifier class must also be provided.

<div class="listbox listbox--no-label">
    <select id="size-selection" aria-label="Choose a size">
        <option value="0" disabled selected>Choose a size</option>
        <option value="1">Small</option>
        <option value="2">Medium</option>
        <option value="3">Large</option>
        <option value="4">Extra Large</option>
    </select>
</div>

Form States

Error State

An error is conveyed using the aria-invalid state. Additional context for the error is conveyed using the aria-describedby property which must point to the unique id of the error message container. This error message container can be designated as a live region if client-side input validation is implemented and enabled.

Please enter a valid email address

<div class="textbox">
    <label for="error-example">Email Address</label>
    <input id="error-example" type="email" name="email" aria-invalid="true" aria-describedby="invalid-email" value="foobar" />
    <div aria-live="polite" id="invalid-email">
        <p>Please enter a valid email address</p>
    </div>
</div>

Disabled State

A disabled field is conveyed using the disabled attribute. Disabled field values are not passed to the server.

<div class="textbox">
    <label for="disabled-textbox">Email Address</label>
    <input id="disabled-textbox" type="text" value="yourname@example.com" name="disabled-textbox" disabled />
</div>
<div class="listbox listbox--no-label listbox--disabled">
    <label for="disabled-listbox" class="clipped">Choose a Size</label>
    <select id="disabled-listbox" disabled>
        <option value="0">Choose a Size</option>
        <option value="1">Small</option>
        <option value="2">Medium</option>
        <option value="3">Large</option>
        <option value="4">Extra Large</option>
    </select>
</div>

Readonly State

A readonly field is conveyed using the readonly attribute. Readonly field values do get passed to the server. Note that the readonly attribute has no effect on a listbox.

<div class="textbox">
    <label for="readonly-example">Email Address</label>
    <input id="readonly-example" type="text" value="yourname@company.com" name="readonly-example" readonly />
</div>

Form Properties

Required Property

A required field is conveyed visually using an asterisk in superscript, and for assistive technology using the aria-required property.

<div class="textbox">
    <label for="email">Email Address <sup>*</sup></label>
    <input id="email" type="email" placeholder="yourname@company.com" name="email" aria-required="true" />
</div>

Toggle Switch

A toggle switch behaves very much like a checkbox - it can be on or off (i.e checked or unchecked) - and under the hood, that's exactly what it is. Whereas checkboxes are often used to allow multi-selection from a group of choices, switches are more often used in isolation or as a series of unrelated options.

<div class="switch">
    <label for="toggle-switch">Returns Accepted</label>
    <input id="toggle-switch" type="checkbox" name="toggle-switch" />
    <span></span>
</div>

Disabled Toggle Switch

A toggle switch can be disabled using the disabled property.

<div class="switch">
    <label for="toggle-switch-disabled">Returns Accepted Disabled</label>
    <input id="toggle-switch-disabled" type="checkbox" name="toggle-switch-disabled" disabled />
    <span></span>
</div>

Icons with Text

Icons with adjacent text are deemed to be non-critical if the text adequately conveys the same message as the icon. Non-critical icons should be hidden from assistive technology using the aria-hidden state.

We had trouble connecting to PayPal. Please make your payment later.

<div class="demo">
    <p><span class="icon-alert" aria-hidden="true"></span>We had trouble connecting to PayPal. Please make your payment later.</p>
</div>

Icons without Text

An icon with no adjacent text may be deemed critical or non-critical to assistive technology, depending on whether it conveys important information or not.

Icons deemed critical must provide alternative text using aria-label and must specify role=img.

<span class="icon-alert" aria-label="Alert" role="img"></span>

Note that in the case of buttons, and other interactive controls, an ARIA state or property might also convey the same information as the visual icon, e.g. a chevron icon and aria-expanded or a sort icon and aria-haspopup, in which case the icon is no longer deemed critical.

Icons without text that are deemed non-critical should be hidden from assistive technology using the aria-hidden state.

<span class="icon-images-null" aria-hidden="true"></span>

Spinner

A spinner animation is used to convey a busy or loading state, often as the result of a user action. A spinner is considered a critical icon and therefore we also convey it's meaning and state to assistive technology.

Small Spinner

By default, all spinners are small.

<span class="spinner" aria-label="Busy animation" role="img"></span>

Large Spinner

Use the spinner--large modifier class to create a large spinner.

<span class="spinner spinner--large" aria-label="Busy animation" role="img"></span>

Menu

A popup menu that allows a user to select from a series of options. A menu can contain button options, radio options or checkbox options.

Menus are typically used to perform AJAX requests and partial page reloads, therefore do not require a form submit button. If you are submitting form data or re-loading the entire page, then a Listbox may be a better choice.

Menu Collapsed

Set aria-expanded state to false on popup button to hide menu.

Tabs

Tabs follow the principal of progressive disclosure by allowing layered stacking of content panels.

When using tabs, you have two choices. Either the tabs are visually styled as tabs, but behave like links (i.e. cause a full page load), or they behave like a tabs widget.

Tabs for Navigation

If you only wish to create a set of links that are styled visually like tabs, with no additional behaviour, please use a simple list of links. No JavaScript behaviour or ARIA attributes will be required.

The tabs__selected class is used to visually denote the current tab/page.

Tabs Widget

For a widget-like experience, whereby selecting a tab will update the panel contents without a full page reload, we can progressively enhance the markup of our navigation tabs example. This time we will be using ARIA roles, states and properties. We must also prevent the default behaviour of the hyperlinks. We can do this by removing the href attribute and specifying role="presentation". In essence, this converts the anchor tag to a meaningless span/div tag.

If you have no interest in using progressive enhancement to create your tabs widget, you can simply replace the anchor tag with a span tag in the example markup below.

Only one tab can be selected and in the tabindex at any moment in time, using aria-selected="true" and tabindex="0". Selecting a tab will deselect all other tabs and remove them from the tabindex. Likewise, only one tabpanel can be visible at any time, using aria-hidden="false", and it must correspond to the currently selected tab.
