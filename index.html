Introduction

Welcome to Skin 2.0. Skin has been rebuilt from ground up to support the new eBay design system named Vision Quest and is not backwards compatible with earlier versions.

Skin 2.0 is the CSS style guide for eBay, which should be used by both engineers and designers. It is in it's purest form a CSS framework with strong focus on accessibility and mobile web. Being exclusively a CSS toolkit means that isolated components that require JavaScript are not part of this project.

This project is operated under an open source model, you can find the code in github. If you wish to contribute feel free to issue a pull request, but we would advise you to talk with us first, either through slack at ebayui.slack.com or by sending an email to DL-ebay-skin@ebay.com. Finally if you want to file a bug, you can do so in the issues section.

What is new in Skin 2.0

Accessibility

Accessibility is at the heart of Skin. A widget must not only look like a certain type of widget, it must be detected and described like that widget too. Skin 1.x relied solely on classes to apply visual styles, but classes do not convey any semantic meaning to assistive technology (or search engines). In Skin 2 we now leverage fully semantic HTML and ARIA roles, states and properties to apply our styles, thus enforcing correct, accessible semantics.

Mobile first

With mobile traffic steadily increasing, we have built this version with a mobile first approach. Modules are compatible with desktop versions so don't be affraid to use them.

Core

The core section of Skin defines boilerplate to centralize common functionality that is re-used by other modules. For the most common use case of skin, which is in its compiled form (.css) it is included a css normalize, base typographic elements and a collection of atomic styles. More advanced users may want to work directly with the .less files, and for those core also provides a collection of usefull mixins.less and variables.less.

Normalize

normalize.css is a customisable CSS file that makes browsers render all elements more consistently and in line with modern standards. Unlike a reset.css normalize.css will not reset browser default styles, it provides instead a default baseline to achieve cross browser consistency. If you want to know more about the differences between normalize.css and reset.css check this answer on stackoverflow.

Typography

Typography creates a unique look for the eBay brand by defining the style of common components links and headers. It provides all the necessary scaffolding for good typography without adding any aesthetics.

Atoms

Atomic CSS is a collection of drop in classes that provide minimal changes to your styles. They are used to extend the CSS properties of individual HTML DOM elements without impacting other DOM elements that share the same base styles. It is also typically suited for rapid prototyping when you want to quickly apply changes without having to modify the underlying CSS.

If used extensivelly Atomic CSS can be an anti pattern in web development. It can lead to readability and maintenance problems, therefore please exercise caution.

Buttons & Links

Buttons are used to trigger form actions (e.g. submit, reset) or JavaScript actions. Links are used exclusively to navigate between destinations, which can be moving from one page to another or to another place in the same page using a hash href="#foobar".

There are two tags for creating buttons: <button> (supports inner HTML) and <input> (supports inner text only). Outside of forms, the <button> is always recommended as it allows the flexibility of nesting useful things, such as icons.

Links always update the URL and therefore the HREF attribute must contain a valid URL. There is only one tag to create links: <a>

Call to action (CTA) links are visually styled like buttons but behave like links. These trigger a GET request, which means that they should always be idempotent. If you wish to trigger an action that chages data on the server, then you should use a form submit to trigger a POST request.

Links are sometimes progressively enhanced into buttons using JavaScript. For example, to view the HREF targetinside of a modal overlay instead of performing a full page navigation. This technique is often reffered to as hijax.

Default Button

To create a Skin Button, use the standard selector .btn. Use for this on all cases including: form actions, primary page actions and any anchor or button element. When using a <button> element, always specify a type. When using a <a> element, always add role="button" for accessibility.

<button class="btn" type="button">Buy it Now</button>

Primary Button

To indicate that a button represents a primary action, add the .btn-prim class name alongside .btn

<button class="btn btn-prim" type="button">Buy it Now</button>

Secondary Button

To indicate that a button represents a secondary action, add the .btn-secondary classname alongside .btn

<button class="btn btn-secondary" type="button">Buy it Now</button>

Active Button

To style a button so that it appears "pressed", add the .active classname alongside .btn or .btn-prim. If aria-pressed="true" is used the button is a toggle button that is currently pressed

<button class="btn active">Button</button>
<button class="btn-prim active">Button</button>

Buttons with different sizes

Small Button

For smaller buttons add .btn-small classname.

<button class="btn btn-small" type="button">Buy it Now</button>

Medium Button

The default is medium size buttons .btn.

<button class="btn" type="button">Buy it Now</button>

Large Button

For larger buttons add .btn-large classname.

<button class="btn btn-large" type="button">Buy it Now</button>

Disabled Button

To mark a button as disabled, add the disabled attribute. A disabled button is unusable and not clickable. To mark a call to action link as disabled, you must remove the HREF attribute.

Disabled 
<button class="btn" disabled>Buy it Now</button>
<a class="btn btn-prim" href="#" role="button" disabled>Disabled</a>
<button class="btn btn-secondary" disabled>Save</button>

Accessible icons

To avoid unintended and confusing output in screen readers (particularly when icons are used purely for decoration), we hide them with the aria-hidden="true" attribute.

If you're creating controls with no other text (such as a button that only contains an icon), you should always provide alternative content to identify the purpose of the control, so that it will make sense to users of assistive technologies. In this case, you could add an aria-label attribute on the control itself.

<span class="icon-cart" aria-label="cart"></span>
<span class="icon-chevron-down" aria-hidden="true"></span>
<button type="button" class="btn btn-default" aria-label="sort">
        <span class="icon icon-sort" aria-hidden="true"></span>
        <span>Sort</span>
    </button>

Tab

Only one tab can be selected at any time, using aria-selected="true". Selecting a tab will deselect all other tabs.

Only one tabpanel can be visible at any time, using aria-hidden="false", and it must correspond to the currently selected tab. Showing a tabpanel will hide all other tabpanels.

Tabs supports progressive enhancement. Before enhancement, the tabs are functional as same page links. After enhancement to tabs (using JavaScript to add ARIA roles, states & properties) the links must be disabled by removing the HREF attribute.

Overlay

A modal displays content that temporarily blocks interactions with the main view of a site. Be sure to add role="dialog" to .modal, aria-labelledby="modal-label" attribute to reference the modal title. When the dialog is in a hidden, closed state you must use display:none to prevent assistive technologies from accessing the overlay.

Mobile Full width Modal

Mobile web modal's are Full page overlays - full page like experience. The width is set to 100%.

Tablet Modal

Set max-width of 800px for the tablet web modals.

Modal Title

Modal content goes here
<div role="dialog" class="modal" aria-labelledby="modal-label" aria-hidden="false">
        <div role="document">
            <h2>Modal Title</h2>
            <!-- dialog content goes here -->
            <button class="icon-close" aria-label="Close Dialog"></button>
        </div>
    </div>
